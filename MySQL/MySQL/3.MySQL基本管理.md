# MySQL基本管理

## MySQL用户管理

```bash
'username'@'host'
host 可以使用通配符 % _
    'shay'@1'0.0.0.%'

查看用户
    select user,host from mysql.user;


创建用户
    #这种方式创建的用户默认权限usage
    create user 'shay'@'10.0.0.%' identified by  'passweord'  

用户重命名
    rename user old_user_name to new_user_name;

删除用户
    drop user 'shay'@'10.0.0.%'

修改密码
    set password for 'root'@'localhost'='123456'
    alter user 'test'@'10.0.0.0.100'  identified by 'password'


刷新权限
    flush privileges;
```

### 用户授权
```bash
授权
    #给wordpress账号授权wordpress库的所有权限
    grant all on wordpress.*  to 'wordpress'@'10.0.0.%';

取消授权
    revoke all on *.*  from 'user1'@'10.0.0.1';

查看用户获得授权
    show grants for 'user'@'10.0.0.1'
    show grants for current_user()
```


## 存储引擎

### MyISAM 引擎特点

```bash
不支持事务
表级锁定
读写相互阻塞，写入不能读，读时不能写
只缓存索引
不支持外键约束
不支持聚簇索引
读取数据较快，占用资源较少
不支持MVCC（多版本并发控制机制）高并发
崩溃恢复性较差
MySQL5.5.5 前默认的数据库引擎

MyISAM 存储引擎适用场景

只读（或者写较少）
表较小（可以接受长时间进行修复操作）


MyISAM 引擎文件
tbl_name.frm 表格式定义
tbl_name.MYD 数据文件
tbl_name.MYI 索引文件

```

### InnoDB 引擎
```bash
行级锁
支持事务，适合处理大量短期事务
读写阻塞与事务隔离级别相关
可缓存数据和索引
支持聚簇索引
崩溃恢复性更好
支持MVCC高并发
从MySQL5.5后支持全文索引
从MySQL5.5.5开始为默认的数据库引擎

```

## 管理存储引擎
```bash
查看mysql支持的存储引擎
show engines;

查看当前默认的存储引擎
show variables like '%storage_engine%';

设置默认的存储引擎
vim /etc/my.cnf
[mysqld]
default_storage_engine= InnoDB


查看库中所有表使用的存储引擎
show table status from db_name;


查看库中指定表的存储引擎
show table status like  'tb_name';
show create table tb_name;

设置表的存储引擎：
CREATE TABLE tb_name(... ) ENGINE=InnoDB;
ALTER TABLE tb_name ENGINE=InnoDB;
```
